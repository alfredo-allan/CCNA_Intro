6.0.1

Por que devo cursar este módulo?

Bem-vindo à camada de link de dados! Toda rede tem componentes físicos e mídia conectando os componentes. Diferentes tipos de mídia precisam de informações diferentes sobre os dados para aceitá-los e movê-los através da rede física.

Analogia: Uma bola de golfe se move rápido pelo ar, mas não tão rápido na água. A camada de enlace é a "ajuda" que os dados precisam para trafegar em diferentes meios físicos.

6.1.1

A Camada de Enlace

A camada de enlace de dados do modelo OSI (Camada 2) prepara os dados da rede para a rede física. É responsável pela comunicação da placa de interface de rede (NIC).

Acesso à mídia

Camadas superiores não precisam saber o tipo de mídia

Encapsulamento

Pacotes L3 → Quadros L2

Troca de quadros

Entre endpoints na mesma mídia

Detecção de erros

Rejeita quadros corrompidos
💀 Aplicação Cybersecurity: Ataques de ARP Spoofing exploram a camada de enlace para associar um endereço MAC falso a um IP legítimo, permitindo interceptação de tráfego (MITM). Impacto: roubo de credenciais e dados sensíveis.
6.1.2

IEEE 802: Subcamadas LLC e MAC

A camada de link de dados IEEE 802 divide-se em duas subcamadas:

Subcamada Padrão Função
LLC
(Logical Link Control)		 IEEE 802.2 Comunica entre software de rede (camadas superiores) e hardware. Identifica qual protocolo L3 (IPv4/IPv6) está sendo usado.
MAC
(Media Access Control)		 802.3, 802.11, 802.15 Implementada no hardware (NIC). Responsável por encapsulamento, endereçamento físico e controle de acesso à mídia.

Funções da Subcamada MAC

  • Delimitação de quadros - Sincronização entre transmissor e receptor
  • Endereçamento - Endereços MAC de origem e destino
  • Detecção de erro - Trailer com CRC/FCS
  • Controle de acesso à mídia - Gerencia comunicação half-duplex 
# Estrutura de um quadro Ethernet (Camada 2)
+----------+----------+----------+-------------+----------+
| Preâmbulo| MAC Dest | MAC Orig | Tipo/Comp. |   FCS    |
| (7 bytes)| (6 bytes)| (6 bytes)| (2 bytes)   | (4 bytes)|
+----------+----------+----------+-------------+----------+
| <-- LLC (802.2) pode adicionar DSAP/SSAP/Control --> |
+-----------------------------------------------------+
6.1.3

Fornecimento de Acesso ao Meio Físico

Cada ambiente de rede tem características diferentes. Um roteador executa estas funções da Camada 2 em cada salto:

1 Aceita um quadro de um meio
2 Desencapsula o quadro (remove cabeçalho L2)
3 Encapsula novamente o pacote em um novo quadro
4 Encaminha o novo quadro para o próximo meio
Cenário Real: Roteadores de borda corporativos frequentemente convertem quadros Ethernet (LAN) para HDLC ou PPP (WAN serial). Uma configuração errada pode causar dropping de pacotes e indisponibilidade de rede. Impacto: perda de R$50k/hora em operações críticas.
6.1.4

Protocolos da camada de enlace não são definidos por RFCs (IETF), mas por organizações de engenharia:

Resumo: Encapsulamento na Prática

PC1 (192.168.1.110) --[Ethernet]--> Roteador --[Serial/PPP]--> Roteador --[Ethernet]--> Servidor (192.168.1.5)

Fluxo do quadro:
1. PC1 enquadra pacote IP com MAC destino do roteador (Ethernet)
2. Roteador desencapsula, remove quadro Ethernet
3. Roteador encapsula em quadro PPP/HDLC (serial)
4. Próximo roteador desencapsula e reencapsula em Ethernet
5. Servidor recebe quadro Ethernet original com MAC correto

# Comando para ver estatísticas de encapsulamento:
Router# show interfaces serial 0/0/0
  Encapsulation PPP, LCP Open, IPCP Open
  CRC errors: 0, frame errors: 0
6.2.1

Topologias Físicas e Lógicas

A camada de enlace de dados precisa conhecer a topologia lógica de uma rede para determinar como transferir quadros entre dispositivos. A topologia de uma rede é a organização e o relacionamento dos dispositivos e suas interconexões.

Topologia Física

Identifica as conexões físicas e como os dispositivos são interconectados. Inclui localização específica como número da sala, rack e prateleira.

Exemplo: Sala do servidor: Sl:2158 · Rack 2 · Bastidor 1

Topologia Lógica

Refere-se à maneira como a rede transfere quadros de um nó para o próximo. Identifica conexões virtuais usando interfaces de dispositivo e endereçamento IP.

Exemplo: Rede 192.168.100.0/24 · VLANs · Rotas
💀 Visão de Cybersecurity: A topologia lógica é o que um invasor enxerga primeiro (sub-redes, VLANs, rotas). A topologia física revela pontos de acesso físico e riscos de interceptação de cabos ou inserção de dispositivos maliciosos (Raspberry Pi, keyloggers físicos).
6.2.2

Topologias WAN

As WANs são interligadas usando três topologias físicas principais:

Ponto a Ponto

Conexão permanente entre dois pontos finais. Topologia WAN mais simples e comum.

Estrela

Site central conecta filiais via links ponto a ponto. Filiais não trocam dados entre si sem passar pelo central.

Malha

Alta disponibilidade. Todos os sistemas interconectados. Custos administrativos e físicos elevados.

Topologia WAN Vantagens Desvantagens Aplicação
Ponto a Ponto Simples, baixo custo, privacidade Sem redundância, falha única Link entre matriz e filial
Estrela Centralizado, fácil gerenciamento Ponto central crítico Redes de franquias/bancos
Malha Redundância total, alta disponibilidade Custo elevado, configuração complexa Backbones, provedores
Cenário Real: Ataques DDoS em topologia estrela podem derrubar o hub central, isolando todas as filiais. Bancos adotam malha parcial para garantir continuidade operacional mesmo com 2-3 links comprometidos.
6.2.3

Topologia WAN Ponto a Ponto

Conexão direta entre dois nós. Não precisam compartilhar o meio físico com outros hosts. Protocolos como PPP (Point-to-Point Protocol) simplificam a camada de enlace. 

# Configuração PPP em roteadores Cisco
Router(config)# interface serial 0/0/0
Router(config-if)# ip address 192.168.1.1 255.255.255.252
Router(config-if)# encapsulation ppp
Router(config-if)# ppp authentication chap
Router(config-if)# no shutdown

# Verificar vizinho PPP
Router# show ppp neighbor
Vantagem: Não há necessidade de determinar destino do quadro. Todos os quadros trafegam apenas entre os dois nós.
Risco: Link ponto a ponto sem criptografia (PPP padrão) pode ser espionado (sniffing). Usar PPP com CHAP/MPPE ou migrar para VPN.
6.2.4

Topologias LAN

LANs multiacesso interconectam dispositivos usando topologias estrela ou estrela estendida. Switches Ethernet são os dispositivos centrais.

Estrela

Dispositivos conectados a um switch central

Estrela Estendida

Múltiplos switches interconectados

Barramento (Legado)

Cabo coaxial, terminadores

Anel (Legado)

Token Ring, FDDI
📡 Legado: Topologias barramento (Ethernet coaxial) e anel (Token Ring) são obsoletas. Rede barramento: se um ponto falha, toda a rede cai. Anel: interrupção em qualquer nó quebra o anel.
6.2.5

Comunicação Half-Duplex e Full-Duplex

Half-Duplex

Dispositivos podem transmitir e receber, mas não simultaneamente. WLANs e hubs Ethernet usam half-duplex. 

# Verificar modo half-duplex
Switch# show interfaces fa0/1
  Half-duplex, 10Mb/s

Full-Duplex

Transmissão e recepção simultâneas. Switches modernos e NICs Ethernet operam em full-duplex por padrão. 

# Verificar modo full-duplex
Switch# show interfaces fa0/1
  Full-duplex, 100Mb/s, media type is 10/100BaseTX
⚠️ Problema comum: Mismatch de duplex (ex: host em full-duplex, switch em half-duplex) causa colisões tardias, CRC errors e latência. Sintoma: lentidão e perda de pacotes.
6.2.6

Métodos de Controle de Acesso

Acesso baseado em Contenção

Todos os nós competem pelo uso do meio (half-duplex). Apenas um transmite por vez.

  • CSMA/CD - Ethernet barramento/hub
  • CSMA/CA - WLANs (Wi-Fi)

Acesso Controlado

Cada nó tem seu tempo determinado para usar o meio (determinístico).

  • Token Ring - Anel com token
  • ARCNET - Legado
6.2.7

CSMA/CD (Carrier Sense Multiple Access / Collision Detection)

Usado em LANs Ethernet legadas (barramento e hubs). Funciona no modo half-duplex.

1 Carrier Sense - Dispositivo "escuta" a mídia antes de transmitir
2 Multiple Access - Vários dispositivos compartilham o mesmo meio
3 Collision Detection - Detecta colisão comparando dados enviados x recebidos
4 Backoff aleatório - Aguarda tempo randômico antes de retransmitir 
# Verificar colisões em interfaces (hubs)
Switch# show interfaces fa0/1
  Input errors: 0 CRC, 0 frame
  Output errors: 0 collisions, 0 late collisions
💀 Ataque Cybersecurity: Colisão forçada (collision flooding) - Invasor pode gerar colisões intencionais para causar DoS em redes half-duplex, impedindo transmissões legítimas.
6.2.8

CSMA/CA (Collision Avoidance)

Usado em WLANs IEEE 802.11. Em redes sem fio, detectar colisão é difícil (transmissor não escuta enquanto envia). CSMA/CA previne colisões.

Processo CSMA/CA

  1. Dispositivo escuta o canal
  2. Se livre, aguarda tempo aleatório (DIFS)
  3. Envia RTS (Request to Send)
  4. AP responde com CTS (Clear to Send)
  5. Transmite dados
  6. Receptor envia ACK
Proteção: Quadros incluem duração da transmissão (NAV - Network Allocation Vector). Outros dispositivos sabem quanto tempo a mídia ficará ocupada.
Vulnerabilidade Wi-Fi: Ataques de Deauthentication exploram quadros de gerenciamento 802.11 para forçar desconexão e capturar handshake WPA2. 
# Verificar estatísticas CSMA/CA em AP Cisco
AP# show dot11 associations
AP# show dot11 carrier busy

Resumo: Controle de Acesso à Mídia

Característica CSMA/CD CSMA/CA Switched Ethernet
Mídia Cabo coaxial, hub Ar (Wi-Fi) Switch + cabo
Duplex Half-duplex Half-duplex Full-duplex
Colisão Detecta (CD) Previne (CA) Não ocorre

Conclusão: Switches modernos eliminam colisões (full-duplex). CSMA/CD é histórico, CSMA/CA continua relevante para Wi-Fi. Entender esses métodos é fundamental para diagnosticar problemas de rede e projetar LANs eficientes.
